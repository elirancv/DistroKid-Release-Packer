## Documentation Standards

**Format:** All documentation is written in Markdown (`.md` files)

**Naming convention:** Files use `UPPER_SNAKE_CASE.md` format for consistency

**Organization:** Documentation is organized by topic and audience (user guides, development, compliance, etc.)

**Cross-references:** Documents reference each other using relative paths

## Contributing to Documentation

**Guidelines:**
- Follow Markdown formatting standards
- Update this index when adding new documentation
- Keep documentation synchronized with code changes
- Use clear, concise language
- Include examples where helpful

**Workflow:**
1. Create or edit documentation files
2. Update this index if adding new files
3. Test Markdown rendering
4. Submit changes via pull request

See `CONTRIBUTING.md` for detailed contribution guidelines.
